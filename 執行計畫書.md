# 台灣大眾運輸查詢 App — 執行計畫書

> 一站查詢台灣 **公車**、**台鐵/高鐵**、**YouBike** 的跨平台行動應用。
> 前端 Flutter、後端 Golang、資料來源 TDX 運輸資料流通服務平臺。

最後更新：2026-06-18

---

## 1. 專案目標

打造一個讓使用者能即時查詢台灣大眾運輸資訊的 App：

- **公車**：路線查詢、站牌動態到站時間、附近站牌
- **台鐵 / 高鐵**：車站時刻表、車次查詢、票價
- **YouBike**：附近站點、即時可借/可還車輛數

核心價值：**單一 App、即時資訊、定位導向**（以使用者所在位置為中心提供附近運具資訊）。

---

## 2. 技術選型

| 層 | 技術 | 說明 |
|----|------|------|
| 前端 | **Flutter 3.35.3** (Dart 3.9.2) | iOS / Android 雙平台 |
| 後端 | **Go 1.26** | 標準函式庫 `net/http`，後續視需要引入 router |
| 資料來源 | **TDX 運輸資料流通服務平臺** | 交通部官方，OAuth2 client_credentials |
| 後端↔前端 | REST + JSON | 後端做為 BFF，封裝/快取 TDX 資料 |

**為什麼用後端 BFF（Backend for Frontend）而非前端直連 TDX？**
1. TDX client_secret 不可外洩到前端 App。
2. 集中快取，降低 TDX 流量上限壓力（公司級 50 萬筆/日）。
3. 統一資料格式、聚合多運具、減少前端複雜度。

## 4. 後端 API 設計（規劃）

| Method | Path | 對應 TDX 資料 | 狀態 |
|--------|------|--------------|------|
| GET | `/healthz` | — | ✅ 已實作 |
| GET | `/api/v1/bike/stations?lat&lng&radius` | YouBike 站點 + 即時車位 | 🔲 Phase 2 |
| GET | `/api/v1/bus/estimated?stopId` 或 `?route&city` | 公車到站動態 (EstimatedTimeOfArrival) | 🔲 Phase 2 |
| GET | `/api/v1/bus/nearby?lat&lng` | 附近公車站牌 | 🔲 Phase 3 |
| GET | `/api/v1/rail/timetable?from&to&date` | 台鐵/高鐵時刻表 | 🔲 Phase 3 |

設計原則：後端對 TDX 的回應做**欄位精簡與正規化**，前端拿到的是 App 自訂的乾淨 schema，而非 TDX 原始格式。

## 7. 風險與注意事項

| 風險 | 對策 |
|------|------|
| TDX 流量上限 | 後端集中快取（動態類短 TTL、靜態站點長 TTL） |
| client_secret 外洩 | 僅存於後端 `.env`，永不進前端；`.env` 已列入 `.gitignore` |
| TDX 資料延遲/缺漏 | 前端明確顯示「資料時間」與無資料狀態 |
| 各縣市公車資料覆蓋不一 | 以 TDX 為單一來源，逐縣市驗證 |

## 8. 本地啟動方式

### 後端
```bash
cd backend
cp .env.example .env        # 填入 TDX 憑證
go run ./cmd/server         # 預設 :8080
curl http://localhost:8080/healthz
```

### 前端
```bash
cd frontend
flutter pub get
flutter run                 # 連接模擬器或實機
```
